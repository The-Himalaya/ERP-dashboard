AUTHENTICATION & AUTHORIZATION
-------------------------------
  Flow:
    1. User visits /login
    2. LoginPage POSTs {email, password} to VITE_API_URL/auth/login
    3. JWT access_token + refresh_token saved to localStorage
    4. AuthContext provides user object globally via useAuth() hook
    5. ProtectedRoute checks role against allowedRoles prop
    6. Unauthorized roles redirected to their own DEFAULT_ROUTE

  User object: { id, email, role, full_name, department, access_token }

  Roles (7 total):
    executive   → all dashboards
    sales       → /sales
    accounts    → /accounts
    production  → /production
    dispatch    → /dispatch
    operations  → /operations
    quotation   → /quotation

  DEV PREVIEW: VITE_DEV_PREVIEW=true in .env.local bypasses auth entirely,
  loading the production dashboard directly. Set to false for production.

PRODUCTION DASHBOARD — MAIN FEATURE
-------------------------------------
  Purpose: Track FRP (Fiber Reinforced Plastic) manufacturing operations.
  Theme:   Dark (bg #0f1117, amber #f59e0b accent, monospace fonts)

  Tabs:
    1. Production & Finishing  (2 sub-tabs: Production | Finishing)
    2. Dispatch
    3. Stock
    4. Raw Materials
    5. Monthly Overview


SUPABASE DATABASE SCHEMA
-------------------------
  Tables (all have UUID primary key + created_at):

  production_entries:
    date DATE, code TEXT, color TEXT, cover_qty INT, frame_qty INT

  finishing_entries:
    date DATE, code TEXT, color TEXT, cover_finished INT, frame_finished INT

  dispatch_entries:
    date DATE, code TEXT, color TEXT, cover_dispatched INT, frame_dispatched INT

  raw_material_entries:
    date DATE, material TEXT, category TEXT (INWARD|OUTWARD),
    department TEXT (PRODUCTION|FINISHING), unit TEXT, qty NUMERIC

  Note: Derived fields (setQty, weights, etc.) are computed in the frontend
  from productCatalog.ts — they are NOT stored in the database.


DATA FLOW
---------
  1. On dashboard mount → fetch all 4 tables from Supabase
  2. Build derived fields using productCatalog helpers
  3. Store in React state (local cache)
  4. User adds/edits/deletes → optimistic UI update immediately
  5. Async Supabase CRUD in background; rollback on error
  6. On add (production/finishing/dispatch) → fire-and-forget POST to
     Google Sheets webhook (sheetsSync.ts) to sync with Google Sheet

  Stock positions are derived on-the-fly:
    stockCover = Σ(producedCover) − Σ(dispatchedCover)
    stockFrame = Σ(producedFrame) − Σ(dispatchedFrame)
    stockSet   = min(stockCover, stockFrame)
    Status: BACKLOG (set<0), PARTIAL (cover/frame<0), OK (all ≥0)


PRODUCT CATALOG (productCatalog.ts)
-------------------------------------
  Product Types (8): MHC, EMHC, ONGC, CGT, ROUND, DS, RCS, WGC
  Sizes (24): 300×300 mm → 1600×1600 mm, commercial inches, DAI variants
  Capacities (11): 2.5T, 3T, 5T, 12.5T, 25T, 30T, 40T, 60T, 90T, E600, F900
  Weight entries (~27): maps code → {coverKg, frameKg, setKg}
  Raw Materials (23): resins, fiberglass, fillers, catalysts, colors, tools

  Code format: {prefix}_{size}_{capacity.toLowerCase()}
  Example: m_600*600_25t (MHC, 600×600 mm, 25 Tonne)


GOOGLE SHEETS SYNC
-------------------
  Mechanism:
    - Google Apps Script Web App deployed at VITE_SHEETS_WEBHOOK_URL
    - On every INSERT (production, finishing, dispatch), sheetsSync.ts fires
      a fire-and-forget POST with mode: no-cors
    - Apps Script receives the JSON body and appends a row to the
      appropriate sheet ("Production", "Finishing", or "Dispatch")

  Setup steps (one-time):
    1. Open Google Sheets, go to Extensions → Apps Script
    2. Paste the code from google-apps-script/Code.gs
    3. Deploy as Web App (Execute as: Me, Access: Anyone)
    4. Copy the deployment URL into VITE_SHEETS_WEBHOOK_URL env var

  Sheets created automatically:
    - "Production"  : Date, Code, Color, Cover Qty, Frame Qty, Set Qty,
                      Cover KG, Frame KG, Total Weight KG, Timestamp
    - "Finishing"   : Date, Code, Color, Cover Finished, Frame Finished,
                      Set Finished, Timestamp
    - "Dispatch"    : Date, Code, Color, Cover Dispatched, Frame Dispatched,
                      Set Dispatched, Cover KG, Frame KG, Total Weight KG, Timestamp
